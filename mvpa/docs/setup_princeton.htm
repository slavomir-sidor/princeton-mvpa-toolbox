Downloading the scripts

Start by ssh'ing to an opus node. If you're trying to do this on a different computer, read the Accessing the SVN from your workstation section first.

You'll need to decide where you want to store the toolbox scripts. I suggest somewhere in your jukebox lab space. From now on, I will refer to $scriptsdir as a placeholder for this location. We're going to make this directory, change to it, and then check the scripts out from the SVN source code repository into it. Make sure you're in the directory where you want $scriptsdir to reside, e.g. /jukebox/yourlab/yourname/:

svn checkout http://sun/repos/distpat/tags/0.5/mvpa/

 

svn checkout http://sun/repos/distpat/tags/0.5/progress/

 

svn checkout http://sun/repos/distpat/extern/afni_matlab

[If you already have a copy of Ziad Saad's afni_matlab library, the second checkout is a bit unnecessary, but I recommend doing it anyway to ensure that you always have the most up-to-date version.]

Lots of files will stream past, and will now be stored in $scriptsdir/mvpa and $scriptsdir/afni_matlab. If this didn't happen, contact me.

Data files

You're also going to need to store a copy of the necessary data files. Let's call this $datadir. Make sure you're in the directory where you want $datadir to reside, then:

mkdir $datadir

 

cd $datadir

 

cp /jukebox/norman/greg/fmri/tutorial_easy/* .

Unfortunately, this is going to take c. 400MB, until we figure out a better solution.

Paths

Now you need to set your paths, so that when you run matlab, it will know about the toolbox m-files.

The simplest way is to run matlab and type:

addpath $scriptsdir/mvpa

addpath $scriptsdir/afni_matlab

But it will become tiresome to do that every time. One easy solution is create a startup.m that Matlab will run every time it opens, and add the 'addpath' command to that. Here's how to do that:

cd ~

 

mkdir matlab

 

echo "addpath $scriptsdir/mvpa" >> startup.m

 

echo "addpath $scriptsdir/afni_matlab" >> startup.m

This changes directory to your home directory. It creates a matlab directory there, and then creates a file called startup.m, with that pair of lines in it. If you already have a ~/matlab directory, you obviously don't need to create one. If you'd rather avoid the unix voodoo, you can just fire up your favourite text editor (e.g. Emacs or Xemacs) and add the lines to startup.m yourself.

[Note that this ~/matlab/startup.m will run whenever you run Matlab. If you have an old version of the toolbox scripts that you'd rather have as your default path while we're testing things, then just create a special startup.m with these two 'addpath' statements in your $datadir directory. Matlab will only use the $datadir/startup.m when you open matlab in $datadir, otherwise it defaults back to the ~/matlab one.]

 

Accessing the SVN from your workstation

You'll need to download an SVN client that will make it easy to grab the latest version of the toolbox.

Windows – Tortoise SVN

we've had minor problems with the way that this deals with linebreaks, but otherwise it's a really nice interface

Mac – Metissian (command-line only)

doesn't require fink or compiling from source and works fine on tiger, though it does require you to add /usr/local/bin/ to your PATH

to do that, add the following line to .cshrc in your home directory:

setenv PATH "/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/bin"

then type:

source .cshrc

in that window to make sure that the changes take immediate effect

OR

svnX (GUI, but limited functionality)

OR

talk to me if you have Fink and are willing to compile from source, and I'll send details about this

Linux – http://summersoft.fay.ar.us/pub/subversion/latest/

Or, you can head to the SVN client page for more information.

Then, checkout the distpat/tags/0.5/ repositories as above and away you go. Note: you may have to use sun.csbmb.princeton.edu rather than just sun if you're not on opus.

You'll also need to get access to jukebox to get the tutorial_easy data directory somehow, and the paths stuff will be different. Hopefully, everything else should still work.

 

Start your engines

Ok, we're ready to go. Change directory to $datadir, and run matlab. Test that all is well – you should see:

>> help tutorial_easy

[SUBJ RESULTS] = TUTORIAL_EASY()

 

This is the sample script for the Haxby et al. (Science, 2001) 8-categories study. See the accompanying tutorial_easy.htm

Now, load up a web browser, then using File / Open file, open $scriptsdir/mvpa/docs/tutorial_easy.htm

One final thing – things are still in a state of a flux, but I'm going to try very hard to avoid any further substantial changes that you'd notice. In order to ensure that you have the latest version of everything at all times, I would ask that you update your version of the toolbox every time you start a session. To this, just change directory to $scriptsdir/mvpa and type:

svn update

This definitely works on opus (but you'll have to set up your own SVN client to use it on your workstation). You should see some filenames stream past, if any have been changed since you last updated, and then a line like 'Updated to revision ###'. That means you're up-to-date.

Note: doing an update will update all of your documentation, including this document (if it has changed).

Good luck.
