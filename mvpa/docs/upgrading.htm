Princeton multi-voxel pattern analysis – upgrading from the old script

http://www.csbmb.princeton.edu/mvpa

 

This is a list of changes that have been made to the old scripts. It's probably not comprehensive.

The easiest way to understand what's going on is probably to run through tutorial_easy.htm.

·             cell array allows you to store multiple versions of your objects

·             subj.data ® subj.patterns{i}.mat

·             regressors transposed

·             subj.regressors ® subj.regressors{i}.mat

·             subj.runs ® a selector object

·             subj.mask ® a mask object

·             subj.mask.coords and subj.mask.idx have been removed – see 'How do I find the coordinates of active voxels in a mask?' in the 'Howto's / Data structures / Masks' section of the manual

·             subj.mask.vol ® subj.masks{i}.mat

·             accessor scripts – get_mat, set_mat etc. – see tutorial_easy.htm and the 'Data structures' section in the manual

·             renamed many of the functions to be consistent

·             anovaVox.m has been split up into two parts – statmap_anova.m and create_thresh_mask.m. Don't call statmap_anova.m directly – just call feature_select.m and it will call statmap_anova.m for you.

·             zScoreTime.m ® zscore_runs.m

·             AFNItoSubj3.m ® load_afni_pattern.m

·             created load_afni_mask.m so you don't have to call BrikLoad.m directly

·             subj.no_lz_subj_no ® subj.header.id

·             subj.args removed – arguments get passed to the functions directly, and stored in their respective objects in the 'created' field

·             subj.args.fnmBrik removed – just pass in the filenames to load_afni_pattern.m

·             subj.args.anova_conds removed – now, the size of the regressors object that you pass in determines which conditions get used

·             subj.args.pcrit removed – pass in an argument to feature_select.m or create_thresh_mask.m

·             class_args.class split up into class_args.train_funct_name and class_args.test_funct_name

·             class_args.layers removed – you just use class_args.nHidden = 0 to set no hidden layer

·             nminusone.m and nminusone_nopeek.m ® cross_validation.m

·             results.withhelds ® results.iterations

·             results.withhelds(i).out ® results.iterations(i).perfmet

·             renamed lots of the results fields for clarity – see 'Classification / The results structure' in the manual for more information
